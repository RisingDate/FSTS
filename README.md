## 📖 Introduction

**FSTS** is an automated experimental design framework for designing, executing, and analyzing complex multi-agent simulations using Large Language Models (LLMs). Rooted in highly adaptable pipeline generation, FSTS supports wide-ranging scenarios—from simulating the Cuban Missile Crisis and international relations to analyzing the efficiency of smart city delivery ecosystems, digital government personnel, and other generalized scenarios.

**FSTS** draws inspiration from the 'Decision Theater' concept, deconstructing experimental design into three core phases: **Script Orchestration**, **Script Finalization**, and **Actor Generation**. Multi-scenario testing shows that agents generated by this framework can perform in the 'experimental theater' according to the script, reproducing experimental results consistent with real-world situations. The introduction of FSTS not only effectively lowers the barrier to designing social science experiments but also provides scientifically sound decision-making support for policy formulation.

## 🎯 Key Features

- **Automated Script Generation**: Translates natural language requirements into fully structured multi-agent experimental scenarios.
- **Dynamic Actor Modeling**: Readily instantiates tailored agents (e.g., Country A, Country B, Delivery Riders, Citizens, Merchants) with distinct variables, goals, and behavioral logic.
- **LLM as Judge**: Combining single-answer scoring and pairwise comparison LLM Judges effectively suppresses position bias, verbosity bias, and self-enhancement bias during evaluation.
- **Specialized Multi-Agent Roles**:
  - **Generator Agents (AD, ED, RA, SS)**: Responsible for drafting actions, resolving environments, modeling responses, and advancing scenario simulations.
  - **Observer Agents (AD, ED, RA, VarObserver)**: Monitors state transitions, assesses relationships, and tracks customized response variables (like global tension index).
- **Flexible Event & Observation Pipeline**: Built-in `pipeline.py` integrates script generation, script finalization, and actor instantiation seamlessly.
- **Model Usage**: Supports integration with leading Large Language Models (such as GPT-4o, GPT-5-mini, DeepSeek-R1 or newer ones) and custom models.

## 🏗️ System Architecture

FSTS relies on collaborative interactions between several structural components:

### Script Orchestration
- **Responsibility**: Experimental scripts are designed to capture the non-linear evolutionary characteristics of real-world events. Inspired by pipeline mechanisms of task decomposition and hierarchical collaboration in industrial production, this framework proposes a hierarchical experimental script generation architecture. It deconstructs complex experimental design tasks into standardized submodules, mapping macro-level goals to micro-level parameters through multi-agent collaboration.
- **Functions**: Minimizes LLM hallucinations and generates multiple candidate experimental scripts from different perspectives.

### Script Finalization
- **Responsibility**: After obtaining candidate scripts, a unique experimental scheme must be selected for execution. To achieve automation, we employ an LLM as a judge to pick the scripts. However, studies have shown LLMs may exhibit position bias, verbosity bias, self-enhancement bias, and limited reasoning capabilities in judgment tasks, affecting evaluation stability.
- **Functions**: Utilizes an "LLM as Judge" approach to automatically evaluate and filter candidate scripts.

### Actor Generation
- **Responsibility**: Addressing the bottleneck of existing frameworks being limited to shallow interactions and lacking deep causal deduction, the **Actor Factory** aims to build cognitive entities with **deductive reasoning** capabilities. The system transforms general-purpose models into specialized Agents adapted to complex situations through a modular injection mechanism.
- **Functions**: Generates actor Agents capable of performing according to the scripts in the simulated environment.

### Evaluation & Pipeline
- **Responsibility**: Direct the orchestration and evaluation of generated scripts.
- **Functions**: Parse requirements, spawn corresponding logic (`actor_generate`, `script_generate`), run step-by-step loops, and generate markdown reports.

## 🚀 Quick Start

### Requirements

- Python 3.10+
- Access to supported LLM APIs (OpenAI or local deployments like DeepSeek-R1 via Ollama/vLLM)
- Standard scientific packages

### Installation

```bash
# Clone the repository
git clone <repository-url>
cd FSTS

# Install dependencies
pip install -r requirements.txt
```

### Configuration

Modify `config.py` to set up your LLM endpoints and API keys. Open `main.py` and adjust the `MODEL_LIST` / `REQUIREMENT_LIST` to select your active model (e.g., `gpt-4o`) and prompt requirement.

### Running

To execute the main simulation pipeline:

```bash
python main.py
```

## 📊 Workflow

FSTS standard execution steps:

1. **Requirement Parsing**: Understands natural language requests (e.g., "Analyze efficiency of digital government workers" or "Simulate the Cuban Missile Crisis with extreme hostility").
2. **Pipeline Initialization**: `script_generate` creates the blueprint, `actor_generate` spins up customized agents like `CountryAAgent`.
3. **Multi-Agent Simulation**: Agents iterate over the scenario timesteps, utilizing `CommunicationManager` to interact.
4. **Observation & Logging**: Observer agents continually monitor and record specific variables to memory.
5. **Output Generation**: Results, causal events, and logs are distilled and documented using `markdown_saver`.

## 📝 About
This framework provides granular control over emergent behaviors from micro-level agent states, empowering broad simulation domains spanning digital infrastructure modeling, historical conflict reproduction, and complex sociotechnical systems.

This work has not yet fully automated the execution of experiments. Currently, the finalized experimental script must be fed to a Coding model to generate actor Agents that can perform in the specified environment. Meanwhile, evaluating the performance outcomes remains a highly valuable area for future exploration.
